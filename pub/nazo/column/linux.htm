Linuxで使っているコマンドたち

基本的な仲間たち

コマンド 効果
pwd カレントディレクトリを確認する
ls ディレクトリの中身を見る
- ls -a : 隠しファイルまで見る
- ls -l : 時刻や権限などをしっかり見る
cd ディレクトリを移動する
- cd test1 : test1 に移動する
- cd .. : 上のディレクトリへ移動する
touch ファイル名を指定して空ファイルを作成する
- touch 1.txt
cat ファイルの内容を表示。
- cat -n ~/.bashrc : nで行番号を表示。
head ファイルの冒頭を表示する。
- head -n 80 script.py 冒頭80行を表示
- head -c 80 script.py 冒頭80バイトを表示
tail ファイルの末尾を表示する。
- tail -n 80 script.py 末尾80行を表示
- tail -c 80 script.py 末尾80バイトを表示
mkdir ディレクトリを作成する
- mkdir test1 : ディレクトリtest1を作成。
- mkdir -p dir1/dir2/dir3 : -pで深く作る
cp ファイルやディレクトリをコピーする。
- cp 1.txt 2.txt : 1.txtを2.txtにコピー、上書きする
- cp -r test1 test2 : ディレクトリは-rで
mv ディレクトリやファイルを移動する、名前を変える
- mv test1 1_haru : test1 を 1_haruにする
rm ファイルを削除する
- rm 1.txt : 1.txtを削除する
- rm -r test1 : ディレクトリtest1を削除する
- rm -rf : 暗黒魔法！ やらない！ -f 禁止！
find ファイルを検索する。
- find . -name "*..py" 今いる場所以下から .py ファイル検索
which which コマンド名 でコマンド名の場所をチェック

個人的なコマンド（特に自分用のは★付き）

コマンド 効果
alias ~\.bashrc内にあるコマンドの効果を確認
nano Linux内のワープロ。nano ~\.bashrc で設定ファイルをいじる
code . カレントディレクトリをVSCodeで開く
cdh ★Haskell用のディレクトリへcd
runghc name.hs name.hsという名前の.hsをHaskellで処理
ghci Haskellの対話環境を開く
cdwork ★仕事用のディレクトリへcd
python3 script.py ★仕事用のスクリプトの実行
python3 str_script.py result ★result フォルダ内のディレクトリをpython3で処理（resultは.pyへの引数として渡している）
cdsite ★ホームページ用のディレクトリへcd
nazoserver ★ホームページ用の仮想サーバーを立てて開く
openpub (絶対パス) ★nazoserverが動いている時にそこを開く

Gitコマンド

コマンド 効果
git clone URL GitHubなどにあるリポジトリを自分のPCに丸ごとコピーする。
- git clone https://github.com/user/repository.git
：今いるところにrepositoryフォルダを作る。
git status 今の変更状況を見る
git add ファイル名 変更したファイルを「次の記録対象」に入れる」
git commit -m "メッセージ" 変更を記録する
git pull リモート側の最新変更を取り込む。
- StableDiffusionのForgeNeoを最新にする時に使った。
git push 自分の記録をリモート側へ送る。
- 開発の人が使う。自分のホームページもいずれは？
git log 過去の記録を見る
git diff 変更内容を確認する。
git branch ブランチ一覧を見る
git switch ブランチ名 ブランチを切り替える
git switch -c ブランチ名 新しいブランチを作って切り替える

自分は、

git add pub/
git commit -m "Update website"
git push

って感じでやってます。

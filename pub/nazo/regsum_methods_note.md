# RegSum Lab methods note

RegSum Lab に出てくる「総和」は、普通の意味の収束和を拡張して、発散級数にも何らかの有限値を対応させるための手続きです。ただし、すべてが同じ性格のものではありません。

大きく分けると、次のような層があります。

- 部分和を平均・変形して極限を見るもの
- 収束因子を入れて極限を外挿するもの
- 関数として定義したあと解析接続するもの
- モデルを仮定して有限部を取り出すもの

同じ級数に複数の方法を適用して値が一致する場合、その値はかなり安定した解釈を持ちます。逆に値が割れる場合は、「その級数に自然な正則化が一意にある」とは言いにくくなります。

## Cesaro summation

Cesaro 総和は、部分和そのものではなく、部分和の平均を見ます。

級数

```text
a_0 + a_1 + a_2 + ...
```

の部分和を

```text
s_N = a_0 + ... + a_N
```

とすると、Cesaro 平均は

```text
(s_0 + s_1 + ... + s_N) / (N + 1)
```

です。

代表例は

```text
1 - 1 + 1 - 1 + ...
```

です。部分和は `1, 0, 1, 0, ...` と振動しますが、その平均は `1/2` に近づきます。したがって Cesaro の意味では

```text
1 - 1 + 1 - 1 + ... = 1/2
```

と読めます。

これは「振動を平均してならす」タイプの総和法です。成長していく発散、たとえば `1 + 2 + 3 + ...` にはそのままでは効きません。

## Abel summation

Abel 総和は、級数に収束因子 `x^n` を入れて、`x -> 1-` の極限を見ます。

```text
sum a_n
```

に対して

```text
A(x) = sum a_n x^n
```

をまず `0 <= x < 1` など収束する範囲で考え、最後に

```text
lim x -> 1- A(x)
```

を取ります。

たとえば

```text
1 - 1 + 1 - 1 + ...
```

では

```text
A(x) = 1 - x + x^2 - x^3 + ... = 1 / (1 + x)
```

なので、

```text
lim x -> 1- 1 / (1 + x) = 1/2
```

です。

Abel 総和は Cesaro より強いことが多く、振動型の級数に自然です。ただし、途中に特異点がある場合はそのままでは使えません。

たとえば

```text
1 + 2 + 4 + 8 + ...
```

に Abel 的に `x^n` を入れると

```text
1 + 2x + 4x^2 + 8x^3 + ... = 1 / (1 - 2x)
```

ですが、`x = 1/2` に極があります。`x -> 1-` まで進めないので、これは Abel 総和ではありません。

## Borel summation

Borel 総和は、係数をいったん階乗で割って収束しやすくし、それを積分で戻す方法です。

形式的には

```text
sum a_n
```

に対して Borel 変換

```text
B(t) = sum a_n t^n / n!
```

を作り、

```text
integral_0^infinity e^(-t) B(t) dt
```

を見ます。

交代級数や、ある程度 factorial より遅く増える係数に対して有効なことがあります。RegSum Lab では、Borel が成功したときは「収束因子を入れてから戻す」かなり解析的な証拠として扱えます。

ただし、Borel 変換や積分経路に特異点が絡むと、値の選び方が問題になります。ここも万能ではありません。

## Euler summation

Euler 総和は、交代級数に特に強い方法です。

典型的には

```text
a_0 - a_1 + a_2 - a_3 + ...
```

のような交代級数に対して、前進差分を使って値を作ります。

代表例として

```text
1 - 1 + 1 - 1 + ... = 1/2
```

や

```text
1 - 2 + 3 - 4 + ... = 1/4
```

が得られます。

RegSum Lab では、Euler は Abel や Borel と比較するための「交代級数用の安定な基準線」としてかなり便利です。

## Hurwitz / zeta regularization

Hurwitz や zeta regularization は、級数をゼータ関数や Hurwitz ゼータ関数の値として読む方法です。

よく知られている例は

```text
1 + 1 + 1 + ... = zeta(0) = -1/2
```

や

```text
1 + 2 + 3 + 4 + ... = zeta(-1) = -1/12
```

です。

これは普通の和ではありません。`zeta(s)` は本来

```text
zeta(s) = sum 1 / n^s
```

として `Re(s) > 1` で収束しますが、その関数を解析接続して `s = 0` や `s = -1` の値を読む、という操作です。

RegSum Lab の Hurwitz 行は、多項式的・準多項式的な骨格を検出し、それを Bernoulli 多項式や Hurwitz ゼータの値として読む道です。

## Ramanujan summation label

RegSum Lab の `ramanujan` 行は、独立の数値計算法というより、「この Hurwitz/zeta 的な値は Ramanujan 総和としても解釈できる」というラベルです。

多項式型の級数では、Ramanujan 総和・Euler-Maclaurin の有限部・ゼータ正則化が近い場所でつながります。

ただし、Ramanujan 総和にも流派や定式化の違いがあるので、RegSum Lab では「Hurwitz が成功した場合の derived label」として控えめに扱うのがよさそうです。

## Exp cutoff

Exp cutoff は、指数的な減衰因子を入れます。

```text
S(eps) = sum a_n exp(-eps n)
```

を小さい `eps > 0` で計算し、`eps -> 0+` の発散部分をモデル化して、有限部分を取り出します。

たとえば `1 + 2 + 3 + ...` は普通には発散しますが、指数カットオフを入れた和は `eps -> 0` で大きく発散します。その発散部分を引いたあとに残る有限項を読む、という考え方です。

これは物理の正則化に近い感触があります。「発散を含む関数を展開して、有限部を取り出す」方法です。

## Window finite part

Window は、有限個の項をなめらかな窓関数で切り出し、`N` を大きくしたときの成長をフィットして有限部を読む方法です。

RegSum Lab では、Hann や Blackman のような窓を使います。

これは「有限部分和の成長から、発散成分と有限成分を分離する」ための診断的な方法です。かなり実験的ですが、他の手法と一致するときは有用な補助線になります。

## Ensemble

`ensemble` は総和法そのものではありません。

複数の方法が成功したとき、その値の中央値やズレをまとめて、「手法間でどれくらい合っているか」を示す要約です。

数学的な意味では、これは「定義」ではなく「診断」です。

## Geometric continuation

`1 + 2 + 4 + 8 + ... = -1` は、幾何級数の解析接続として読むと自然です。

普通の幾何級数は

```text
1 + r + r^2 + r^3 + ... = 1 / (1 - r)
```

です。ただし、この等式が普通の和として成り立つのは

```text
|r| < 1
```

のときです。

右辺

```text
1 / (1 - r)
```

は、`r = 1` 以外では意味を持つ有理関数です。そこで、収束領域 `|r| < 1` で得られた閉形式を、収束領域の外へ解析接続して読むことができます。

`r = 2` を代入すると

```text
1 + 2 + 4 + 8 + ...  ~~  1 / (1 - 2) = -1
```

です。

ここで `~~` と書いたのは、これは普通の和でも Abel 総和でもなく、「幾何級数の母関数を解析接続して読んだ値」だからです。

RegSum Lab に追加するなら、名前は次のようなものがよさそうです。

```text
geometricContinuation
```

または

```text
generatingFunctionContinuation
```

対象を限定するなら、まずは

```text
a_n = C r^n
```

の形を検出して

```text
sum C r^n = C / (1 - r)
```

を表示するのが安全です。

## 1 / (1 - r^2) は何か

`1 / (1 - r^2)` は、次の幾何級数です。

```text
1 + r^2 + r^4 + r^6 + ... = 1 / (1 - r^2)
```

つまり、偶数冪だけを拾う級数です。

これを係数列として見ると、

```text
1, 0, 1, 0, 1, 0, ...
```

に対応します。

なぜなら

```text
1 / (1 - r^2)
```

を形式的冪級数として展開すると、

```text
1 + 0*r + 1*r^2 + 0*r^3 + 1*r^4 + ...
```

だからです。

この級数を `r = 1` に近づけると発散します。`r = 2` を代入するような解析接続を考えるなら、

```text
1 + 4 + 16 + 64 + ... = 1 / (1 - 4) = -1/3
```

です。

一方で、係数列そのもの

```text
1 + 0 + 1 + 0 + 1 + 0 + ...
```

を「各項の和」として読むと、これは

```text
1 + 1 + 1 + ... 
```

を偶数番目だけに置いたものなので、普通には発散します。Abel 的には

```text
lim x -> 1- 1 / (1 - x^2)
```

で、やはり発散します。

ただしゼータ的・Ramanujan 的に「密度 1/2 の定数列」として有限部を読むなら、別の正則化値が出る可能性があります。ここが重要で、`1/(1-r^2)` の解析接続値と、係数列の総和法としての値は、どの変数を何として読むかで意味が変わります。

## 1 / (1 - r^k)

一般に

```text
1 / (1 - r^k) = 1 + r^k + r^(2k) + r^(3k) + ...
```

です。

係数列としては

```text
n が k の倍数なら 1、そうでなければ 0
```

という周期列です。

たとえば `k = 3` なら

```text
1, 0, 0, 1, 0, 0, 1, 0, 0, ...
```

です。

このような周期列は、RegSum Lab の準多項式・周期検出と相性が良いです。係数列としての総和を考えるのか、`r` に特定値を入れた幾何級数として考えるのかで、別の問題になります。

## 1 / (1 - ar)

`1 / (1 - ar)` は

```text
1 + ar + a^2 r^2 + a^3 r^3 + ...
```

です。

`r = 1` と見ると、

```text
1 + a + a^2 + a^3 + ...
```

になります。

`a = 2` なら

```text
1 + 2 + 4 + 8 + ... ~~ -1
```

です。

この見方では、`a` は列の成長率で、`r` は母関数の変数です。

## 1 / (1 + r)

`1 / (1 + r)` は

```text
1 - r + r^2 - r^3 + ...
```

です。

`r = 1` で読むと、

```text
1 - 1 + 1 - 1 + ... ~~ 1/2
```

になります。

この例は Abel 総和とも一致します。`r = 1` までの道中に特異点がないためです。

## 幾何型を RegSum Lab に入れるときの注意

`geometric continuation` は、かなり自然ですが、万能の総和法として入れると危険です。

安全な実装方針は次のようなものです。

1. 列 `a_n` が `C q^n` に非常によく一致するか調べる
2. 一致する場合だけ `C / (1 - q)` を表示する
3. `q = 1` に近い場合は特異として扱う
4. Abel/Borel/Euler とは別カテゴリとして、`analytic-continuation` または `geometric-continuation` と明示する

たとえば

```text
1, 2, 4, 8, ...
```

なら `C = 1`, `q = 2` なので

```text
C / (1 - q) = -1
```

です。

一方、

```text
1, 0, 1, 0, 1, 0, ...
```

は単純な `C q^n` ではなく周期列です。この場合は `1/(1-r^2)` という母関数はありますが、「幾何型」としてではなく「周期的母関数」や「rational generating function」の扱いになります。

## rational generating function continuation

さらに広げるなら、幾何型だけでなく、有理母関数を推定する方法も考えられます。

たとえば係数列

```text
1, 1, 1, 1, ...
```

の母関数は

```text
1 / (1 - r)
```

です。

係数列

```text
1, 0, 1, 0, ...
```

の母関数は

```text
1 / (1 - r^2)
```

です。

係数列

```text
1, 2, 4, 8, ...
```

の母関数は

```text
1 / (1 - 2r)
```

です。

係数列

```text
1, 2, 3, 4, ...
```

の母関数は

```text
1 / (1 - r)^2
```

です。

この最後の例で `r = 1` を直接代入することはできません。極があるからです。しかし、ゼータ正則化では

```text
1 + 2 + 3 + 4 + ... = -1/12
```

となります。

つまり、母関数の解析接続とゼータ正則化は関係しますが、単純に「母関数に r=1 を代入する」だけでは足りない場合があります。極があるときは、有限部を取るのか、別の正則化を使うのかを決める必要があります。

## まとめ

`1 + 2 + 4 + 8 + ... = -1` は、幾何級数の閉形式を解析接続して読む、正当な形式的・解析的操作です。

ただし、これは Abel 総和ではありません。`1/(1-2x)` が `x=1/2` に特異点を持つため、`x -> 1-` の Abel 極限に到達できないからです。

RegSum Lab に入れるなら、まずは次のように分けるのがよさそうです。

- `geometricContinuation`: `C q^n` 型を検出して `C/(1-q)` を表示
- `rationalGeneratingFunction`: 将来的に周期列や線形漸化式型の母関数を扱う
- `finitePart`: 母関数や cutoff が極を持つ場合に、発散部分を除いた有限項を見る

この分け方にすると、`1/(1-r)`、`1/(1-r^2)`、`1/(1-2r)`、`1/(1-r)^2` が同じ「母関数」という大きな枠に入りつつ、どの値をどう読むのかを混同しにくくなります。

### セオレミウム語文法

### 語順
基本的には SOV が正式な話し方になるのだが、色んな言語をその語順のまま訳せる、詩をその語順のまま翻訳できることを想定しているため、
```Seoremium
Subject :: wi, wi-a            // 主語
Verb2   :: cotolone-ii         // 述語
Object  :: in-tenariima-ni     // 目的語

SOV「wi in-tenariima-ni cotoloneii」
// 私は妹に歌った。

SVO「wi cotoloneii in-tenariima-ni」
// 私は歌った、妹に。

VOS「in-tenariima-ni cotoloneii wi-a」
// 妹に歌った、私は。
```
どれも可という形にしたい。と思うと、負荷は上がるものの形容詞にも格を付す方が更に語順の自由度は上がると思う。とは言い条、やはり単語を修飾する場合、副詞や形容詞はその前後に置くのが望ましいと思う。

#### 名詞
- 性・複数変化はなし！（管理しきれません）
- 事情により、冠詞および格助詞を付す。
- 主格、あるいは体言止め（～は）- φ
    強調したい場合は -[[a]] 
- 属格（～の） - [[is-nun]] （例：私の = [[is]] + [[wi]] + [[nun]] = iswinun）
- 与格（～に） - [[in-ni]] （例：私に = [[in]] + [[wi]] + [[ni]] = inwini）
- 対格（～を） - [[ita-lun]] （例：私を = [[ita]] + [[wi]] + [[lun]] = itawilun）
- 具格（道具・手段・受身の施事） - [[il-rar]] （例：私で、私によって = [[il]] + [[wi]] + [[rar]] = ilwirar） 
- 地格（場所で）- [[ito-dam]]（例：沖縄で = itookinawadam）
- 冠詞 is,in,ita,il,ito については、必ず名詞の直前につく。接頭辞つきの名詞の場合も同じで、「接頭辞つき名詞」の直前につき、格助詞は接尾辞の後につく。
- 要するに、「冠詞・接頭辞・本体・接尾辞・格助詞」が基本的な名詞モジュールで、これは基本的には壊さないようにする。形容詞が付く場合は、名詞モジュールの前後につくのが自然。
- 不定冠詞・定冠詞の区別はないものとする。あくまで格変化の一環。（後々、この辺のilとかinとかが道具感とか場所感を持つようになっていく可能性もありそうだ）
- 冠詞や格助詞を省略すると、ちょっと砕けた表現になる。

#### 形容詞（子音か、i終わり）
- 日本語っぽく「マーカー」を入れて意味を分離していくかも。
    多い [[al]] に対して、大きい [[alki]] とか。
- ラテン語っぽく、格を名詞に合わせることがある。格が多いほど厳格な文体となる。例を挙げる。
    - 「大きい私」の手 = [[alki]]-nun is-[[wi]]-nun [[tan]].
    - 私の「大きい手」 = is-[[win]]-un [[alki]] [[tan]].
- ルール① 『公文書ルール』形容詞には冠詞 is in ita il ito を付けない。ただし名詞化された形容詞『大きさ』などには付けるようにする。
    - メリット：この場合、冠詞の有無により、形容詞と名詞が区別できて便利である。語順が自由になる。
    - デメリット：格がくどい。
     - 私の大きな手が、妹の丸きを、頭を撫でる = Is-[[wi]]-nun [[alki]] [[tan]] is-[[tenariima]]-nun [[mani]]-lun ita-[[tema]]-lun [[nate]].
- ルール②『文語体ルール』形容詞＋名詞を一つの名詞節として、大きく冠詞と格助詞で挟む。
    - メリット：二重主語と思われそうな埋め込み節を、冠詞と格助詞で挟み込むため、パースしやすくなり、どこからどこまでが与格や対格かが分かりやすくなる。冠詞格助詞によるネストも可能になる。
    - デメリット：語順の自由度が落ちる。
    - 私の大きな手が、妹の、丸い頭を撫でる = Is-[[wi]]-nun [[alki]] [[tan]] is-[[tenariima]]-nun ita-[[mani]]-[[tema]]-lun nate.
    - 私は妹がいることを必要とする。Wi ita-[[tenariima]]-[[jeve]]-[[ata]]-[[iputo]]-lun une.
- ルール③『詩文・口語ルール』名詞においても冠詞を（リズムに応じて）省略する。形容詞においては冠詞のみならず、格助詞まで語感やリズムを重視して省略してもよい。
    - メリット：語順が自由になる。
    - 私の大きな手が、妹の丸い頭を撫でる。Wi-nun alki tan tenariima-nun mani-tema-lun nate.
    - 私の大きな手が、妹の頭を……丸い……撫でる。= [[Wi]]-nun alki [[tan]] [[tenariima]]-nun [[tema]]-lun [[mani]] nate.
- ルール④『現代詩・緊急ルール』すべての冠詞も格助詞も省略してしまう。もっとも砕けた言い方になる。
    - メリット：とにかく速い。
    - デメリット：意味が不明確になるため、短い文しか言えない。
    - 私、小さい妹、撫でる。[[Wi]] [[chalki]] [[tenariima]] [[nate]]. 

#### 動詞 （原形は-eで終わること多い）
- 時制マーカー
    - 現在・原形（辞書形） -φ-
        - いる。（jeve）
    - 過去 -ii-
        - （その時に）いる。（[[jeve]] + -i = jeveii）
    - 未来 -oi-
        - いる予定。（[[jeve]] + -oi = jevoi）
- 能動態・受動態マーカー
    - むずかしいので「される」みたいにマーカーを入れる。
        - 単純な受動態は -ra- とする。
            いられる。jevera
            いられた。jeve-ii-ra
        - 中動態は -tosal - とする。
            なんとなくいることになる。jevetosal
        - 検討した結果、使役はこの能動・受動マーカーでは表示しない（埋め込み節の増加による組み合わせ爆発を経験したことが理由）。代わりに、後述の不定法マーカー [[-iputo]]を用いて ita-動詞-iputo-lun + [[se]] という形で、使役動詞を用いる。これは後述する助動詞の仲間だとも言える。例文を挙げると、
            私は彼にいさせた。Wi in-ariig-ni ita-jeveiputo-lun se.
            彼は私にいさせた。Ariig in-wi-ni ita-jeveiputo-lun seata.
        - 使役動詞は単独の動詞なので、受動態が入れる。
            彼は私にいさせられた。Ariig in-wi-ni ita-jeveiputo-lun seraata.
- アスペクト(進行、持続、完了、経験、将然(いままさに))
    - 進行／持続（今まさにしているところだ）-tin
        - 居続けている。（[[jeve]] + tin = jevetin）
    - 完了（した。それから今までそうである）-to
        - 今までいた。（jeve + -to = jeveto）
    - 経験（したことがあり、今はそうでない）-lutip
        - （その時は）いたことがある。（jeve + -lutip = jevelutip）
    - 将然（今まさに、まもなくするところだ）-kum
        - （今まさに）いるところだ。（jeve + -kum = jevekum）
- 否定マーカー
    - -[[nek]]-
        - 居続けない。（nek+jeve+tin = nekjevetin）
- 法
    - 不定法 -iputo（動詞を名詞化するため、後ろに格助詞を付属可能）
        - いること（jeve + -iputo = jeveiputo）
        - 発展形に、「ということ」 [[-siputo]] も。（[[sa]]+[[iputo]]）
        - 居続けないこと（jeve-tin-nek-iputo）
        - 居続けないということ（jeve-tin-nek-siputo）
        - 「なに、俺の運命がテメエを… デリートるって事だったって事だよ！」（フリゲ『メイジの悲劇嘆』より）\*（非文マーカーを付す）
          （Def, winun seiputo uslun... eevene-ata-siputo-siputo-naru!）\*          
    - 接続法は今のところなし（というか私がわからない）
    - 命令法は……語尾 [[-mei]] などでカバーする。
- 人称マーカー
    - 非人称・辞書形 -∅
    - 一人称単数 -∅（ただし非人称・辞書形と区別するために、-ne -zo -naru -ta -yolなど語尾を付けることが多い。付けない時は文語的になる）
    - 二人称単数 -su
        - あなたがいること（jeve + -su + -iputo = jevesuiputo)
    - 三人称単数 -ata
        - 彼がいる（Ariig jeveata）
        - 彼がいた（Ariig jeve-ii-ata）
        - 彼が居続ける（Ariig jeve-tin-ata）
        - 彼が居続けないこと（Ariig nek-jeve-tin-ata-iputo）
    - 一人称複数 -mus
        - 私たちがいること（Witan jeve-mus-iputo）
        - 私達がいるということ（jeve-mus-siputo）
    - 二人称複数 -ti
        - あなたたちがいること（Ustan jeve-ti-iputo）
        - あなたたちがいたということ（Ustan jeve-ii-ti-siputo）
        - あなたたちがいます（Ustan jeve-ti-dus）
    - 三人称複数 -nata
        - 彼らがいること（Ariigtan jeve-nata-iputo）
    - 連用形 -artas（同主語の別動詞が続く場合用いる）
        - 彼らはいて、笑う。（Ariigtan jeve-artas ahhane-nata）
        - ariigtan jeve-nata, ahhane-nata.でも良し。
        - 「彼らはいて、ぼくは笑う」の場合は使えない。その場合はariigtan jeve-nata, wi-a ahhane.

ある程度崩れてもいいのだが、正式な口調のマーカーの順序としては、以下のようになる。（`NSTAVoPeMo`、みたいに覚えるかもね）

```PlainText
Neg × Stem × Tns × Asp × Voice × Psn × Mood
```

- 否定 - `Neg :: pos, neg`  （肯定、否定）
- 語幹 - `Stem`（変わらない部分）
- 時制 - `Tns :: prs, pst, fut`（現在、過去、未来）
- アスペクト - `Asp :: prog, pft, exp, immin`
  （進行、完了、経験、将然）
- 態 - `Voice :: act, pass, mid`（能動、受動、中動）
- 人称 - `Psn :: imp, 1sg, 2sg, 3sg, 1pl, 2pl, 3pl`
  （非人称、一単、二単、三単、一複、二複、三複）
- 法 - `Mood :: ind, inf, claus`（直説法、不定法、名詞節的）

という感じ。

例：
	Is-wi-nun tenariima in-paja-ni nek-jeve-ii-tin-ra-ata-iputo-cuna?
（私の妹が、父に居続けられなかったことかな？）
```Seoremium
   Is-Wi-nun 
// My

   tenariima 
// younger sister

   nek - jeve  - ii  - tin  - ra    - ata - iputo - cuna?
// neg - exist - pst - prog - pass  - 3sg - claus - Q
// Neg - Stem  - Tns - Asp  - Voice - Psn - Mood  - Tmnt
```

Us hiun indamni jeveiilutipsucuna?
- あなたはそこにいたことがあるの？
> us あなたは
> hiun その
> in-dam-ni 場所に（与格）
> [[jeve]] 存在する
> ii 過去（存在した）
> lutip 経験（存在した経験がある）
> su 人称（二人称単数）
> [[cuna?]] 思い惑う疑問（のかな？）

#### 世界のズレ、バグなどについて
これらは、「異文化」「異世界」などを無理やり合わせた時に生じる「ズレ」を説明するための語彙である。

以下の「ズレをあらわす要素」から「シーム値」として整数値で計算したり、あるいは「翻訳不可能要素」のリストとして計算されたりと、色々な型によって明示化される。

セオレミウム文化はこうしたことを面白がり、かつなんとかしようとする。あらゆるズレは一つのパラメータのみで計れることではなく、いくつかのパラメータによって計測される。

##### （ちょっと高級にすぎる）準備
- ひとつの（到達不能、存在不能=`leko`な）事実世界トポスを $\mathcal{W}$ とする。
- 概念同士の射をもつ小さな圏 = `seoremium` を作品世界 $\mathcal{V}$ とする。
- 作品世界の実装 `enesaw`（＝嵯：創語、プログラム）は、$F : \mathcal{V} \rightarrow \mathcal{W}$ とする。
- 各人のトポスを $\mathcal{W_i}$ と置き、幾何的射 $p_i : \mathcal{W_i} \rightarrow \mathcal{W}$ を「私→事実世界」とする視座とする。
- 部分対象分類子 $\Omega$ が何を真とみなすか、Lawvere-Tierney位相が「何を覆うか」は、各人のトポス $\mathcal{W_i}$ によって異なる。
- 4人称「inusni toropanun wi - sulwa」＝あなたにとっての私は、基底変換 $(-)_{p-\text{us}} \circ (-)_{p-\text{wi}}$ を意味する。
- 分数人称は、他者のトポスから自己トポスへと「真」を引き戻す作用である。
-  `sanatan` / 物語合意＝視座を関手で集約し、余極限（pushout）を取るイメージ。
- 世界の遷移を時間推進自己関手 $T$ と余代数$(X, \delta : X \rightarrow TX)$ で表現する。

##### 対応の観点（意味・翻訳）
###### `tivhaze` - 型ズレ（Scheme Mismatch, SM）
- 定義：両世界A/Bの概念グラフ（ノード：概念、エッジ：関係）を最適対応付けしたときの、グラフ編集距離÷節点数。
- より高級に：関手 $F : \mathcal{V} \rightarrow \mathcal{W}$ が充分忠実でも、本質全射でもない割合のこと。編集距離は、「像でない成分の大きさ／射の折りたたみ量」を表す。
- 直感：概念の対応のズレ。
- 例：`ecosaae` 「床屋に入ったと思ったら遷移先が初期位置だったせいで海に放り出されたオジサンの話」。
    - 現実世界は「遷移先」とかではなく局所的にユークリッド空間として看做せる座標で動くのだが、それをあえてゲームっぽくしてふざける。
- 例：ゲーム「設計される側の世界=`seoremium`」の語彙と「実装する側の語彙=`enesaw`（＝プログラム）」のズレ。
- 例：バグったアイテム
    - 「このアイテム欄の`i'pima` ってなんだ？」（名称＝実体型）
    - 「それ、バグ道具（＝`hazetan`）だわ……。多分アイテム名のポインタがグラフィック定義か何かを読んでる（＝`tivhaze`）んじゃない？ 昔のゲームだからメモリ領域が使い回されてるんだよね……ちなみにそれどうやって手に入れたの？」
    - 「アイテム交換のためにセレクト押して、何も交換せずにそのまま閉じて、戦闘中に魔法でセレクト押したら道具欄がバグっちゃって……（魔法と道具の交換？ 型制約の破綻＝`tivhaze`）」
    - 「なるほど、想定してない交換のせいでメモリが壊れて、アイテム欄が広くなっちゃったのかな……」
- 型解析：
    - `Tool :: Entity -> Bool`
    - `Item :: Ord -> String`
    - `ToolUse :: Tool -> State(Seoremium)`
    - `ItemUse :: Item -> ItemEffect`
    - `ToolQuantity :: Tool -> Int`
    - `ItemQuantity :: Item -> Int`
    - `Knapsack :: [(Tool, ToolQuantity)]`
    - `ItemSlot :: [(Item, ItemQuantity)]` 
    - 基本的には、`Tool <-> Item, ToolUse <-> ItemUse, ToolQuantity <-> ItemQuantity, Knapsack <-> ItemSlot` という形で翻訳が上手く行っている。
    - のだが、「実装されてない」アイテムIDに対しても、`Ord` が本来のアイテム名領域をはみだして参照、メモリテーブルと関数ポインタによってアイテム名、効果、価格などが対応してしまい、売買できたり、使用してバグ効果を呼び出せる。
    - 本来 `Ord` は世界内の `Entity` に1対1対応すべきだが、実装上は単なるID参照なので、定義済のアイテムID以外の順序数においては、世界観と乖離する（`Tool <-> Item`の崩壊）。これが `tivhaze` の典型。この場合、『存在しない道具ノード』が余計に増えており、対応が取れない＝編集距離が大きい。
- ズレの種類
    - 概念欠落型（片方に存在しない）
    - 粒度差型（Aでは1語、Bでは複数語ある）
    - 意味転倒型（真逆に対応してしまう）
- 測り方（簡易）
    - A/Bで同義語辞書を作り、ハンガリアン法で対応→未対応ノード比＋関係の断絶を合算。
    - 代替：A語彙→B語彙の機械翻訳精度（F1やBLEU）を1−で取る。
- 下げて安定化するためには？
    - 語彙を増やす／中間語彙（メタモデル）をかませる。
- 上げて演出するためには？
    - あえて何も噛ませない。

#### 瀬語の不可能人称・多重人称・分数人称・小数人称について
##### 不可能人称

セオレミウム語には例えば、us（あなた）、ariig（彼）という単語や、-us, -ataという人称マーカーがあり、  us jeve-su（あなたがいる）や、ariig jeve-ata（彼がいる）といった文がある。  
  
こうしたus 〇〇-su, ariig 〇〇-ataは「可能人称」を表す標示である。  
  
「可能人称」という言葉について整理がいる。  
セオレミウム語における「あなた=us」「彼=ariig」とは、  
「私という主体にとってのあなた・彼」という意味でしかなく、  
超越的に「あなた・彼」そのものを指すことはない。  
  
us hakile-suは、「私にとって、あなたは走っている」といった意味であり、ariig jeve-hakile-ataは、「私にとって、彼は走っている」といった意味となる。 ある種の「独我論的言語」だということが出来るかもしれない。  
  
つまりセオレミウムにおける「us」像とは、sefwar（≒自我）という嵯語（輸入された高位語）を用いれば「到達可能なiswinun us（私から見たあなた）」と「到達不可能なisusnun sefwar（あなたの自我）」の二つの「部分us」の直和（非交和）だと考えることができ、また後者は実質言及不可能な「不可能人称」となる。  
  
もちろん、「あなたの自我」には「あなた」の言葉によってある程度近づくこと自体は可能であり、その時、「isusnun sefwar」は「iswinun us」、ひいては「私＝wi」により近づくことになる。  
  

##### 多重人称

以上のような「不可能人称」という領域を考慮する時、「私から見たあなたから見た私」というものは、当然本来の一人称とは別れることとなる。これを、「あなたのあなた」、2^2=4人称と考えることが出来る。  
  
「あなたにとっての私」という（主語）表現は、in-ni toropanun（～にとっての）とus（あなた）とwi（私）という三つを組み合わせて、「inusni toropanun wi」という風に瀬語で表すことが出来る。  
  
これに動詞 jeve を与える場合、通常の通り「inusni toropanun wi jeve」とすることも可能だが、「inusni toropanun wi jeve-sulwa」という人称マーカーを付けることが可能である。これは「四人称文」であることを明示する「多重人称調・多重人称文体」である。  
  
他にも六人称を表す、  
「あなたにとっての彼=inusni toropanun ariig」と「sulata」、  
「彼にとってのあなた=inariigni toropanun us」と「atalsu」といった組み合わせや、  
九人称を表す「彼にとっての彼女=inariigni toropanun ariima」と「ataluta」といったものがある。  
これらは意味は違うが、どちらも（あくまで指標上）六人称である、と整理される。  
  
こうした喋り方は、「あなたにとっては～だけど、私にとっては～」という逆説＋抗弁調の表現だけではなく、  
「あなたにとっては～だったんだね、だから～」という順説＋理解、「inusni sefwar」への接近とも出来る。  
こうした世界観はセオレミウムびと特有の「多重化・分散化した自己や時間」を創り出していく契機となるのだが……。  
  

##### 分数人称

「本来、実質不可能な二人称・三人称＝us, hiun」や  
「四人称的な自己＝inusni trospanun wi-sulwa」といった人称を、  
唯一実在する一人称・「wi」に近づけるための思考法として、分数人称というものがある。  
これはある種の熟慮、あるいは逆説的忘我の過程なのではないだろうか？  
  
```
wi --id--> wi
|           |
2     ┘    id
|           |
v           v
us --1/2--> wi
```

  
wiがある限り、usに辿り着くことは出来ない。  
しかし、wiがないと、wiに辿り着くことは出来ない。  
よって、「wiがあり、かつない」というある種のメタ視点を見ていくことになる。  
これが「1/2人称」の視座となる。2人称を1人称に引き戻すような視座のことだ。  -teul、という人称で表現される。(ita-ten-us-lun、くらいの感じ)
itawilun zaileteul.（私は自身を、あなたの目線で見つめる）
itawilun zailoiteul.（私は自身を、あなたの目線で見つめ直すだろう）

同様に、（非実在）n人称を（実在）1人称に引き戻す視座を、1/n人称と呼ぶ。1/3人称（-teatal）とか、1/4人称（-tesulal）とか言うのはレア。
itawilun zaileteatal.（私は自身を、あの人の目線で見つめる）
itawilun zailoiteatal.（私は自身を、あの人の目線で見つめ直すだろう）
itawilun zailetesulal.（私は自身を、あの人から見た私の目で見つめ直すだろう。→私は「あの人に期待された自分」と自分自身とのズレを見つめ直す、みたいな意味）

また、この視座から多重化されたm人称は、m/n人称とされ、それは、よくわからないものとされる（？）  
  
##### 小数人称

それはそれとして、「us」と「ariig」のような異なる対象の間に共通する、中点的なイメージ（あるいはハイパータイプ的な抽象）を定義することが出来る。これに特段2.5人称とかと名を付けることはないのだが（有理数に限る必要はなさそう）、とにかく小数人称と呼ぶ。

##### まとめ
- 二、三人称 
    - あくまで一人称の主体から見た物しか見えない
- 多重人称
    - なることの不可能な「他者から見た自分や他者」を想像して語る
- 分数人称
    - 想像されたn人称を自分ごとに引き戻す話法

#### 同調現象
以下は「セオレミウム」という架空の地で起きる（かもしれない）特異現象について述べる。

ある主体は、彼から見た「n人称」には到達不能なのだが、仮に（限定的な場面などで起きやすい）到達したと互いに信じられた場合に同調現象は起きる。同調した二者は視点がメタへと移り、二つの肉体を（融合した）一つの意識で操ることになる。初めは支障が大きいが、慣れるとすごく便利。遠隔地においてまったく時間差なく情報を共有できるため、「同調びと」は電話としても有用である。

同調現象は、cotolone(共同演奏)の場で起きやすい(そのため意図して起こすために演奏技術を磨く場合がある)。

また、同調現象は、サビラ語に比較して発達した人称マーカーと、古サビラ語を組み合わせて呪文として発動する場合がある。こちらは演奏のパターンに比べ、より自由に同調現象をセッティングして扱うことができ、相手が望んでいなくても無理やり無生物と同調させるなども可能なため、意図した発動は所管行政庁の免許を要する独占業務となる。

#### 『親の魔法』
また、同調現象は、年齢差のある対象に対して、『親の魔法』として発動される場合がある。これは同調と言えば聞こえはいいが、（まだ一人称の未発達な）子（＝esham）を一方的に従属させることを言う。他の世界観だと『ギアス（制約呪文）』と呼ばれるだろう。『親の魔法』とは言い条、技術次第で血縁関係なしに用いることが可能である。

- 発動中及び解除後相当期間持続する、「同調先への強い愛着=hatene」
- 発動中持続する「自己拡張感と甘い快感」
- 解除時に「胸が張り裂けるような人生最大の失恋」のような感覚
- 解除後も持続する「多重人称・分数人称・ズレ計測能力の鈍化」

などの重い副作用が伴う。そのため、ほんの短時間であっても強靭な精神的抵抗力を必要とし、予後としても「恋愛ができなくなる」などの副作用が伴うことがある。そもそも子の自由を奪う行為であることもあり倫理的な問題が大きいため、厳格に制限されている。古サビラ語の知識のある者によって（幼児の）人命救助などに緊急的かつ瞬間的に用いられることが多くその場合は罰則も生じないが、上記の副作用からは逃れられない。そのため、『親の魔法』を専門に扱う者たちの間では、後述の「無生物との同調」を行う者が増える傾向にある。

- 父性属性の親の魔法は強権的に禁止と命令を行う。子の身体操作権限を奪取する。
- 母性属性の親の魔法は条件文、未来形、願望、取引と相性が良く、依存関係の条件的肯定や否認を伴う。子の精神性へ影響する。

これでは聞こえが悪いが、実のところ社会（象徴秩序）によって似たことは（成人に対してですら、法律といった形で）日常的に行われており、必ずしも悪い面ばかりではない。そもそも育児自体が呪文を伴わぬ軽い『親の魔法』の一形態であるとされており、また良い父性、良い母性というものに決まった形はない。むしろ、「悩んでいること」こそが良い親の証拠とされるのである。

セオレミウムびとによる育児は、『親の魔法』と同様、終了時にやはり精神的抵抗力次第では強い喪失感を覚えることがあり、そうした親は、机やぬいぐるみなどといった「無生物との同調」によってリハビリをするようになる。だが昨今は親としての期間を飛び越えて直接「机期」に移行する者も増えており、早晩セオレミウムは「家具文明」に移行するのではないかと目されている。


#### 無生物との同調
無生物との同調は、人との同調に比べれば遥かに容易であり、そこら辺の子供であってもぬいぐるみ相手に用いる。精密な熟達のためには繊細な動作描写語彙や条件文を要する。このために古サビラ語を学習することもある。悪用厳禁。無生物との同調には二方向あり、

- マスター型：無生物を意のままに動かす（ぬいぐるみを可愛く動かしたり）
- スレーブ型：無生物を模写する（上手ければ無生物そのものになりきれる）

となっている。スレーブ型を上手く使って、敵襲を机のふりで乗り切るとか、体の中に何かをしまう、などはよくある話である。


#### 架空概念との同調
こちらは無生物であったとしてもより高難度。`seoremium`（架空話、仮想世界）から持ち帰った魔具（光の矢を打ち出す剣など）を `seoremino`と呼び、慣習的にその呼称は`《★ekus-ekus》` といった記号で飾られることが多い。

また、不可能世界から持ち帰った物質（四角い円など）を `seolekino` と呼び、こちらには`《⊥ekus-ekus》`といった記号が付される 。`seolekino`の方が一般的に危険を伴う。バグ道具（＝`hazetan`）を持ち込んだりして混乱を招くと罰を与えられる。上記の『親の魔法』の濫用者も同様だが、罰を受けている者は、同調試験による反省が認められない限りは`coremetosaliputo`などでの発言の信頼性が低下するため、その期間は代わりに机などとして活用される。（机びとからの非正規的な発言が飛び交うことは多く、にぎやかしやランダム要素として`coremetosaliputo`に取り入れられることはある）

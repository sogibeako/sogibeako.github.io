以下は、「GTX 1070で、Gitも黒い画面もわからない人が、Stable Diffusion WebUI Forge Neoを入れようとして、黒い窓に脅されながら最終的に動いた」くらいの導入メモである。

Stable Diffusion / Forge Neo ざっくり導入メモ

0. これから何を入れようとしているのか？

Stable Diffusionは、PC上で画像生成するための仕組みである。ただし素のままだと扱いにくいので、普通はブラウザで操作できる画面、つまりebUIを使う。

WebUIの有名どころはこんな感じ。

名前 ざっくり
AUTOMATIC1111 昔から定番のStable Diffusion WebUI
Forge A1111系をベースに、軽量化や最適化を狙った派生
Forge Classic 古めのForge系を安定寄りに保つ版
Forge Neo Forge Classic系の別ブランチで、より新しめに更新されている版

今回はForge Neoを導入する。 Haoming02/sd-webui-forge-classic の neo ブランチとして配布されており、GitHub上では2026年5月時点でも更新が続いている。元のForgeはAUTOMATIC1111系を土台に、開発しやすさ・リソース管理・推論速度などを意識したWebUIである。

1. GTX 1070で最初に知っておくこと

GTX 1070はPascal世代の少し古いGPUであり、生成のためにはネックとなる。最近のPyTorch / CUDA環境では、GTX 1070などのsm_61世代が対応外になっていることがあり、

GTX 1070 is not compatible with the current PyTorch installation
みたいなエラーで止まる場合がある。（というか実際止まった）。

そのため、動かすなら、古めのPyTorchに固定することが必要となる。

2. 事前に入れるもの

Python公式では3.11系のWindows向けリリースが配布されている。Git for Windowsも公式から最新版を入れられる。uvはAstral公式がWindows向けのインストール方法を案内している。

3. フォルダを作る

まず、変な場所に置かないのが大事。

ChatGPTによるとおすすめは C:\ForgeNeo とのことだったが、個人的にはもう専用のSSDに置いてしまうのがいいと思う。内蔵出来ない場合もあると思うので、外付けでもいいかも（自分としては外付けの G: 直下に入れている。十分動いている）。

日本語フォルダ、OneDrive配下、深すぎるフォルダ、スペース入りフォルダは避けたほうがいい。Forge Neo導入記事でも、最初は C:\ForgeNeo のような単純な場所が無難とされているようだ。

例：

mkdir G:\ForgeNeo
    cd /d G:\ForgeNeo

PowerShellでこれを入力する。

  • mkdir：make directory。フォルダを作る。
  • cd /d：change directory。指定したアドレスのフォルダに入る。/d は別のドライブに移動する場合のオプション。

せっかくなので黒い画面の人達について

  • cmdはWindowsの古めの黒い画面。StableDiffusionとかLameとかで開く感じの画面。dir, cd, copy, del みたいな画面を使う。.bat と仲がいい。
  • PowerShellは、Windowsの新しめの黒い画面。今回のStableDiffusionのインストール作業ではこれを使うよ。pwd, cd, ls のようなLinuxっぽいコマンドにも対応しているけど、完全なLinuxとはちょっと違う。
  • Git Bashというのもある。これはGit for Windowsについてくる、Linuxっぽい黒い画面。WindowsでGitをコマンドラインから使うためのBashエミュレーション。コマンドはls, cd, pwd, cp, rm, git clone とLinux寄り。Windowsのパスも、Git Bashでは /c/ForgeNeo という感じでLinuxっぽく見る。導入しておくと、エクスプローラー右クリックで「Git Bash Here」が使えるようになる。
  • WSLは、Windows上でLinuxを動かす仕組み。導入しておくとUbuntuとかも使える。（注意：今回は使わないよ！ 特に、こっちで git clone しちゃうと、Linux側（私の場合は /mnt/... みたいなところに保存されてしまいます）
  • 開発ではあまり混ぜないのが無難。Gitを入れるなら、Windows側、Ubuntu側、それぞれに入れるイメージになる。

4. Forge Neoを取ってくる

Forge Neoは、Forge Classicのneoブランチとして入れる。同じくPowerShellで、

git clone https://github.com/Haoming02/sd-webui-forge-classic sd-webui-forge-neo --branch neo
    cd sd-webui-forge-neo

意味はこうだ。

  • git clone ＝ GitHubからフォルダごとコピーしてくる
  • --branch neo ＝ neo版を指定する（ForgeNeoが欲しいので）
  • cd ＝ そのフォルダに入る。上でもやったね。

「黒い画面怖い！」と思うかもしれないが、だいたいはファイルを落としているだけなので驚くことはない。

5. uvを入れる

PowerShellで以下を実行する。

powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

これでuvというPython環境作成ツールを入れる。もし入ったか確認したければ、

uv --version

でチェック。

これで uv が見つからないと言われたら、PowerShellを一度閉じて開き直す（まだ認識されてないだけかも）。

6. Python仮想環境を作る

Forge Neoのフォルダ内へとcdで移動していって、

uv venv venv --python 3.11.9 --seed

これは、このForge Neo専用のPython部屋を作ってねという意味。Python環境はPC全体をぐちゃぐちゃにしないための小部屋で、その部屋の名前が venvだ。

venvの使い分け

  • 本格的に何かを動かすプロジェクト
    ：プロジェクトごとにvenv
  • Stable Diffusion / Webアプリ / 依存関係が重い
    ：必ず専用venv
  • ちょっとしたPython練習・一発スクリプト
    ：共通venvでもまあOK

もし何も考えず、PC全体のPythonに pip install していくと、だんだんと危険になっていく。それぞれ別の工具箱のイメージなので、Stable Diffusion用、音楽スクリプト用、Webアプリ用、と別にしていくのがいい。

venvごとに、pipで入れたライブラリを分けてくれる。

  • numpy
  • torch
  • flask
  • django
  • pillow
  • requests

みたいに色々あるんだけど、その中でも「torch 2.4.1」とか「torch 2.8.0」とか「numpy 1.26」とか「numpy 2.0」とかあって、しかもそのライブラリ同士に相性がある（ヒィー）

例えばStable Diffusionでは、torchはこれ、xformersはこのtorchと合わないとダメ、CUDAはGPUに依存してるのでそこからtorchが自然と決まる、Pythonも新しすぎるとダメ……。

だから、PC全体のPythonにpip installしていくのはマズいん。「どのバージョンの何がどれに関わってるの？」となっていくので。あるプロジェクトのために pip install --upgrade したら、他のプロジェクトが動かなくなるかも……（ヒィ～～）

7. webui-user.batを編集する

Forge Neoフォルダの中にある、webui-user.bat を右クリックして、メモ帳で開く。（短いのでメモ帳で良し。あるならVSCodeで開くとカッコイイ）

たとえば以下のような行を探そう。

set COMMANDLINE_ARGS=

GTX 1070なら、まずは軽め設定で、

set COMMANDLINE_ARGS=--uv --xformers --medvram

ただし、GTX1070だと xformers が逆にトラブルになることもある。その場合は外そう。それでももし起動しなければ、さらに

set COMMANDLINE_ARGS=--uv --lowvram
  • うちのComfyUIはこれじゃないと動かない。

8. 初回起動する

webui-user.batをダブルクリック。

黒い画面が出る。めちゃくちゃ文字が流れて、ここで不安になるかもしれない。

Installing...Downloading...Collecting...Building...

そしてめっちゃ時間がかかる。だけど、これは基本的には正常だ。別にウィルスを落としているわけではなく、必要な部品を入れているだけ。

最後にブラウザで、

http://127.0.0.1:7860

のような画面が開けば成功。

127.0.0.1は「自分のPC自身」という意味で、インターネットのどこかに飛んでいるわけではない。自室の中に小さな画像生成工房が生えた状態だ。だから、思う存分、君のイマジネーションをぶつけよう。

おっと、Modelsフォルダの中の \StableDiffusion の中に、お好みのモデル（.safetensor形式が安全。.ckptは任意コード実行が指摘されている）を入れていこう。 \VAE の中にいい感じのものを入れるのも。

9. GTX 1070で詰まったところ

CUDA capability sm_61が対応外

NVIDIA GeForce GTX 1070 with CUDA capability sm_61 is not compatible with the current PyTorch installation

今入っているPyTorchが、GTX 1070を古すぎるGPUとして見ているのだ。対応の方向性は、

venv\Scripts\activate.batpip uninstall torch torchvision torchaudio -ypip install --index-url https://download.pytorch.org/whl/cu121 torch==2.4.1 torchvision==0.19.1 torchaudio==2.4.1

GTX 1070などPascal世代では、古めのPyTorch、たとえば torch==2.4.1 + cu121 に戻す回避策が報告されている。

10. 初回生成のおすすめ設定

GTX 1070なら、最初は控えめにしよう。興味が出てきたら、Hires.FixとかControl NetとかADetailerとかLoRAとか導入すりゃいいのです。LoRAは悪名高い絵師絵柄LoRAばっかりじゃなくて、色んなシチュエーションとかもあるので。

項目 おすすめ
モデル SD1.5系（一応SDXLも動く！ 遅いけど）
Width 512
Height 512
Steps 20
Sampler Euler a / DPM++ 2M Karrasあたり
CFG Scale 5〜7
Batch size 1
Hires.fix 最初はOFF

SDXLはキャラをだいぶ学習している（特にnoob系とかはすごいよ。でも絵柄的にはillustriousとかmellow_pencilとかが個人的に好きなんだな）

まあ、まずはとりあえず上の設定で

1girl, glasses, beret, cozy room, soft light

みたいな短いプロンプトでテスト。

ネガティブプロンプトは

low quality, worst quality, blurry

くらいでOK。Generateを押すと、画面でジワジワとキャワイイ女の子が生成されてくることだろうよ……黒い画面でも|

100%|████████████████|

みたいなかっこいい感じのバーが伸びていく……。これかっこいいよね……。

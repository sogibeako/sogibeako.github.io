# ブラウザゲーム用 モンスター＆スキルデータ仕様書

本ドキュメントは、モンスターの行動やスキルを定義・管理するためのデータ構造（CSVファイル/シート定義）をまとめたものです。
最終的にこれらの表データを JSON に変換し、ゲームのプログラム側で読み込んで使用することを想定しています。

## 構成シート一覧

1. **monsters** (モンスター基本情報)
2. **tag_defs** (種族・属性タグの定義)
3. **monster_action_rules** (モンスター別行動ルール)
4. **skills** (スキル詳細)
5. **skill_sets** (スキルセット定義)
6. **skill_ai** (AI用スキル評価)
7. **targeting_rules** (ターゲット選択ルール)
8. **traits** (特性定義)

---

## 1. monsters (モンスター基本情報)
モンスターの土台となる能力値（Lv1時の初期値と成長率）や持つタグ、判断力などを定義します。

| 列名 | 内容 | 記述例 |
| --- | --- | --- |
| `id` | 内部ID | `fire_slime` |
| `name` | 表示名 | 炎のスライム |
| `vitBase`, `vitGrowth`| 生命力/VITの初期値と成長値（HP・防御に影響） | 15, 2.0 |
| `intBase`, `intGrowth`| 知力/INTの初期値と成長値（MP・魔法力に影響） | 8, 1.0 |
| `strBase`, `strGrowth`| 腕力/STRの初期値と成長値（物理攻撃力に影響）| 20, 3.5 |
| `dexBase`, `dexGrowth`| 器用さ/DEXの初期値と成長値（命中・クリティカルに影響）| 10, 1.5 |
| `spdBase`, `spdGrowth`| 速さ/SPDの初期値と成長値（行動順・回避・多段に影響）| 12, 1.2 |
| `judge` | 判断力(AIの賢さ, 0～100など) | 35 |
| `tags` | カンマ区切りのタグ群 | `"slime,fire"` |
| `defaultTargetRule` | 基本の狙い方ID | `normal_enemy` |
| `note` | メモ | Lv40から炎魔法 |

## 4. skills (スキル詳細)
個々の技や魔法の性能を定義します。回復だけでなく、与えたダメージを吸収する「ドレイン仕様」や、変則的な魔法効果を含みます。

| 列名 | 内容 | 記述例 |
| --- | --- | --- |
| `skillId` | スキルID | `drain_touch` |
| `name` | スキル名 | エナジードレイン |
| `type` | 攻撃種別(magic/physical/special等)| `magic` |
| `targetType` | 対象(single_enemy等) | `single_enemy` |
| `element` | 属性 | `dark` |
| `power` | 基礎威力 | `80` |
| `costMp` | 消費MP | `8` |
| `drainHpRatio`| 与ダメージの何%を自分のHP回復にするか | `1.0` (100%吸収) |
| `drainMpRatio`| 与ダメージの何%を自分のMP回復にするか | `0` |
| `formula` | 適用する計算式タイプ | `mag_based` |
| `formulaParam1, 2` | 計算式に渡すパラメータ | `0.25`, `999` |
| `specialEffectId` | 特殊例外処理呼び出し用ID | |
| `note` | メモ | 魔法扱いのHPドレイン |

### ※ `type` (スキル種別) の役割について
`type` はゲームの基本ルールの判定（計算の手前）に使われます。
* **`magic` (魔法)**: 「沈黙」状態にされると使えなくなる。「魔法反射」の対象になる。
* **`physical` (物理特技)**: 「沈黙」関係なく使える（吸血や体当たり、剣技など）。「暗闇」状態で命中が下がったり、「回避」パラメータで避けられたりする。
* **`special` / `breath` (特殊能力やブレス)**: ブレスやアイテムなど。沈黙でも使えて、物理回避によって避けられない（必中）という処理によく使われます。敵の「自己再生」や薬草による回復は魔法ではないため `special` にしておくと沈黙で封じられません。
| `targetType` | 対象(single_enemy等) | `single_enemy` |
| `element` | 属性 | `dark` |
| `power` | 基礎威力 | `0` |
| `costMp` | 消費MP | `8` |
| `formula` | 適用する計算式タイプ | `current_hp_ratio` |
| `formulaParam1, 2` | 計算式に渡すパラメータ | `0.25`, `999` |
| `specialEffectId` | 特殊例外処理呼び出し用ID | `boss_cap_999` |
| `note` | メモ | 現在HPの25%ダメージ |

## 6. skill_ai (AI用スキル評価)
自動戦闘でAIが「今この技を使うべきか」を採点するための指標です。

| 列名 | 内容 | 記述例 |
| --- | --- | --- |
| `skillId` | スキルID | `fire` |
| `baseScore` | 基本点 | `50` |
| `preferWeakness` | 弱点を突ける相手への加点 | `40` |
| `preferKill` | これで倒せそうな時の加点 | `25` |
| `preferLowHpAlly` | 味方HPが低い時の回復加点 | `60` |
| `preferNotApplied` | 状態異常未付与時の加点 | `40` |
| `uselessIfAlreadyApplied`| 状態異常済みなら致命的減点するか | `true` |
| `mpPenalty` | MP消費量に対する減点評価 | `10` |

| 列名 | 内容 | 記述例 |
| --- | --- | --- |
| `targetRuleId` | ルールID | `brute_force` |
| `preferWeakness` | 弱点持ちを狙う比重 | `0` |
| `preferLowHpEnemy`| 瀕死の敵を狙う比重 | `30` |
| `preferLowDefEnemy`| 防御が低い敵を狙う比重| `20` |

## 実装上で大事なポイント

1. **データとプログラムの役割分担**
   すべての処理を表やデータだけで表現しようとせず、表計算データはあくまで「設計図・パラメータ」に留めます。
2. **変わった特技の組み方**
   スキルテーブルの `formula` (基本計算式: `atk_based`, `current_hp_ratio`等) と `specialEffectId` (例外的な追加効果処理: `double_if_poison` 等) の2種類をプログラム側に渡すことで、複雑な魔法も「基本式＋例外処理」として整理できます。
3. **判断力 (Judge) を活かした AI**
   AIの基本点に、`skill_ai` による「状況評価ボーナス(弱点、瀕死優先など)」を加算します。
   モンスターの `judge`（判断力）が高いほど、最も点数の高い的確な行動を選びやすくなり、頭の良い敵と本能で動く敵の差別化がデータ上で可能になります。

## 各種基礎パラメータ（ステータス）の定義と戦闘への影響（案）

「VITでHPを決める」「INTでMPを決める」といった本格的なRPGの構造を採用する場合、ステータスは**「キャラクターの肉体的な特徴（基本ステータス）」**と、そこから計算される**「戦闘用の数値（派生ステータス）」**という2段構えに分かれます。

### 1. 基本ステータス（BaseとGrowthを持つ大元の数値）

* **VIT (Vitality / 生命力)**
  * **役割**: タフさを表します。主に **最大HP** と **防御力(DEF)** の計算基準になります。
  * **副効果**: 毒や麻痺などの「状態異常にかかりにくさ」に影響させても面白いです。
* **INT (Intelligence / 知力)**
  * **役割**: 頭の良さや魔法の素質。主に **最大MP** と **魔法力(MAG)** の計算基準になります。
  * **副効果**: ステータス異常の付与成功率を上げたり、回復魔法の効果量をさらに底上げすることがあります。
* **STR (Strength / 腕力)**
  * **役割**: 物理的な力。**攻撃力(ATK)** の計算基準になります。
* **DEX (Dexterity / 器用さ)**
  * **役割**: 手先の器用さ、正確さ。
  * **クリティカル率**: 弱点を正確に狙い撃つため、会心の一撃が出やすくなります。
  * **命中率**: 攻撃を外さない正確さを表します。
  * **盗む・逃げる**: 固有スキルの成功率に直結させることができます。
  * **特定武器の威力**: 「弓」や「短剣」のような武器は、STRではなくDEXを攻撃力の基準にすることが多いです。
* **SPD (Speed / 速さ)**
  * **役割**: 行動の速さ。
  * **行動順**: ターン中の動く順番。
  * **回避・多段**: 回避率や、相手より一定以上素早い場合の「多段ヒット（連撃）」の判定に用います。

### 2. 派生ステータス（プログラム側で戦闘時にVITなどから計算される数値）

上記の基本ステータスから、装備品の補正などを足し合わせて以下の数値を弾き出し、実際のダメージ計算式などで使います。

* **HP (Hit Point)**: 例 `VIT × 5 + レベル補正 + 基本HP` 
* **MP (Magic Point)**: 例 `INT × 3 + 基本MP`
* **ATK (Attack)**: 物理攻撃力。例 `STR × 1.5 + 武器の攻撃力`
* **DEF (Defense)**: 物理防御力。例 `VIT × 1 + 防具の防御力`
* **MAG (Magic)**: 魔法力・魔法防御。例 `INT × 1.5 + 装備の魔力`

このようにデータを設計することで、「HPがやたらと高いが防御力は紙（スライムなど）」を作るか「VITが高くてHPも防御力をもガチガチ（ゴーレムなど）」といった深い表現ができるようになります。

現在位置 : ホーム ＞ 実装位相論について

非対称隣接空間と主体依存位相
——rotによる空間の分裂

Asymmetric Adjacency Spaces and Agent-Dependent Topology: Spatial Splitting Induced by Rotation State

水原（Mizuhara）

実装位相論ノート 第2稿

概要

ファミコンRPGの2Dタイルマップは、一見すると単純な有限格子に見える。しかし実装の観点から見ると、重要なのは「点がどこにあるか」そのものではなく、各マスからどこへ移動できるか、画面端でどのようにラップするか、向きや反転・回転状態がどう保存・変換されるかといった遷移規則の方である。

前稿（「実装位相論：計算機上の空間構造に関する数学的基礎」）では、この立場を離散ファイバーバンドルとホロノミーの言葉で定式化した。本稿では、その枠組みの中で特に回転状態 rot がもたらす現象——隣接関係の非対称性、主体ごとに異なる位相空間、接触空間と描画空間の分裂——を分析する。

具体的には、位相ローグライク topologue_v05.htm の実装を読解し、そこで起きている数学的構造を明らかにする。

キーワード：非対称隣接、主体依存位相、rot、ホロノミー変数、接触空間・描画空間の分裂、離散曲率

1. 空間ではなく遷移系として見る

1.1 真の状態空間

通常、2Dマップは座標集合 $X$ の上で考えられる。しかしゲーム内部では、座標だけでは状態を記述しきれない。プレイヤーの向き、左右反転・上下反転の有無、回転状態、季節や位相モード、乗り物状態、世界の表裏——これらのすべてが移動結果に影響する。

そのため、真の状態空間は

$$ \mathcal{S} = X \times F \times Q $$

となる。ここで：

  • $X$：マップ座標（ノードID）
  • $F$：向き・反転・回転などの内部位相状態（前稿の topoState）
  • $Q$：季節、世界位相モード、ボス生存フラグなどの大域状態

各移動方向 $d \in \{\uparrow, \downarrow, \leftarrow, \rightarrow\}$ に対して遷移写像

$$ T_d : \mathcal{S} \to \mathcal{S} $$

を定義することで、世界全体をひとつの離散力学系として扱える。

1.2 大域整合性の条件

このモデルが数学的に整合するためには、以下の条件が必要である。

  1. 一歩移動が well-defined：どの状態からどの方向へ進んでも、遷移先が一意に定まる。
  2. 経路合成が well-defined：2歩、3歩、1周と移動を合成した結果が、写像の合成として定まる。
  3. 世界全体が統一的に読める：局所的な接続変更やイベント変化があっても、それらが全体としてひとつの規則系に収まる。

前稿で示した離散ファイバーバンドルの枠組みは、この3条件を自然に満たすモデルであった。

2. 位相の動的変化：空間が呼吸する世界

前稿では、接続規則（ラップ命令）は静的に与えられていた。しかし実際のゲームでは、接続規則が時間やイベントに応じて変化することがある。ここでは、動的な位相変化の類型を整理する。

2.1 季節・状態で変わる境界接続

通常時は東端から西端へそのまま出るが、冬だけ南北反転して出る：

  • 通常：$(W-1, y) \mapsto (0, y)$
  • 冬：$(W-1, y) \mapsto (0, H-1-y)$

見た目は同じマップでも、冬だけメビウス的なねじれが生じる。前稿の言葉では、大域状態 $Q$ に「季節」を含め、季節によって接続の遷移写像 $g_e$ が切り替わるモデルである。

2.2 ボス生存時のみ非可換な世界

東端通過で回転、北端通過で反転を導入すると、「先に東へ行くか、先に北へ行くか」で結果が変わる。

$$ g_E \circ g_N \ne g_N \circ g_E $$

この非可換性は、ボスを倒すと $g_E$ が恒等写像に戻って可換になる——という形で、「呪われた世界」の解呪として実装できる。

2.3 周回で変わるダンジョン

塔や迷宮が周回ごとに接続規則を変える：

周回

ダンジョンそのものが、位相を状態とする有限状態機械として振る舞う。これは前稿§6の位相遷移力学系の具体例でもある。

2.4 時間でラップ先がずれる世界

東端通過先が時間 $t$ に応じて

$$ (W-1, y) \mapsto (0,\ y + k(t) \bmod H) $$

のように変化する。$k(t) = \lfloor t / 100 \rfloor \bmod H$ とすれば、100ターンごとにラップ先が1マスずつずれていく。「空間が呼吸する」「世界そのものに潮汐がある」ようなモデルである。

3. ラップ命令への干渉

3.1 特殊タイルが局所接続演算子になる

神殿の床や虹の橋など、特定地点だけ別の接続規則を持たせることができる：

  • 北へ出ると180度回転して南から出る
  • 通過時に世界層（表/裏）が反転する
  • 橋を渡ると表世界から裏世界へ切り替わる

これはタイルが単なる床ではなく、局所接続演算子になっている例である。前稿§4の「局所接続」の具体的実装に相当する。

3.2 アイテムによるラップ規則の改変

「裏返しの杖」「空間ねじれの宝玉」などを使うと、境界ラップ規則に変換が加わる。flipYを付加する、rotを加算する、ラップ先オフセットを反転する——いずれも前稿の定理4.13（局所変換によるホロノミー変化）の直接的な実装になる。

3.3 範囲外参照からの干渉

マップ外へ出た結果、意図しないメモリ領域を踏み、そこに置かれていた「境界接続テーブル」が書き換わる。プレイヤー体験としては「外海の果てで禁断の敵を倒したら、帰還後に世界の継ぎ目がおかしくなった」という神話的現象になるが、実装的にはテーブル破壊——前稿§5のフラグマイニングそのものである。

3.4 マップ切り替えを貼り合わせとして見る

町、フィールド、塔内部がそれぞれ別の位相を持ち、出入口はその間の貼り合わせ写像になっている。「マップ切り替え」は単なる場面転換ではなく、局所チャートの貼り合わせとして読める。これは多様体論の座標変換（チャートの遷移関数）の離散版である。

3.5 主体ごとに異なる空間

プレイヤー、NPC、亡霊、船などで使う接続規則を変えることもできる。「プレイヤーには行けない道を亡霊が歩く」「老賢者だけ別位相の道を見ている」「船だけ海上トーラスを移動できる」。これは主体依存空間であり、本稿後半の主題に直接つながる。

4. rot の正体：ホロノミー変数としての回転状態

本章から、topologue_v05.htm の具体的な実装を読解し、rot の数学的意味を明らかにする。

4.1 rot は向きではなく、境界通過履歴の蓄積である

QuarterTurnTopology は、端を抜けるたびに接続先の辺を90度回転させる位相である。その resolveStep の核心部分を見る：

// QuarterTurnTopology.resolveStep() より抜粋
const toEdge = this.turn === "clockwise"
  ? rotateDir(fromEdge, 1)
  : rotateDir(fromEdge, -1);

({ x: nx, y: ny } = this.placeOnEdge(toEdge, t, width, height));

nextTopoState.rot =
  this.turn === "clockwise"
    ? (nextTopoState.rot + 3) % 4
    : (nextTopoState.rot + 1) % 4;

境界を抜けるごとに rot が変化する。rot は「今どちらを向いているか」ではなく、どんな境界の貼り合わせを経て今ここに来たかを記録している。したがって rot は、前稿の定式化におけるホロノミー変数そのものである。

4.2 入力と描画は逆向き変換を使う

Topologue では、プレイヤー入力と局所描画がきれいな対を成している：

処理
関数
rot の使い方
入力変換
mapInputDirToGraphDir()
$\text{rot}$ で順方向に回転：主観→世界
描画変換
projectDirToLocalOffset()
$-\text{rot}$ で逆方向に回転：世界→主観

入力変換は「プレイヤーが『上へ行こう』と思った方向を、現在の rot に応じて世界グラフ上の方向に変換する」処理である：

function mapInputDirToGraphDir(inputDir, topoState) {
  let dir = inputDir;
  if (topoState?.flipY) {
    if (dir === "N") dir = "S";
    else if (dir === "S") dir = "N";
  }
  if (topoState?.flipX) {
    if (dir === "E") dir = "W";
    else if (dir === "W") dir = "E";
  }
  dir = rotateDir(dir, topoState?.rot ?? 0);
  return dir;
}

一方、描画変換はその逆操作——世界方向を主観座標系へ引き戻す：

function projectDirToLocalOffset(dir, topoState) {
  let localDir = rotateDir(dir, -(topoState?.rot ?? 0));
  // flipX, flipY の逆適用...
  switch (localDir) {
    case "N": return { dx: 0, dy: -1 };
    case "E": return { dx: 1, dy: 0 };
    // ...
  }
}

この対称性は、ファイバーバンドルにおけるフレームの持ち上げ（入力）と射影（描画）の関係に対応する。

5. 真世界と主観空間の二層構造

5.1 buildWorldGraph() は静的な客観世界を作る

世界グラフは buildWorldGraph() で構築される。ここで決定的なのは、隣接を構築するときに常に topology.initialState()——つまり rot = 0——を使っていることである：

for (const dir of DIRS) {
  const result = topology.resolveStep(
    node.x, node.y, dir,
    topology.initialState(),  // 常に初期状態
    baseMap.width, baseMap.height
  );
  node.neighbors[dir] = { to: toNodeId, transform: result.transform };
}

したがって worldGraph は、動的な rot を持たない初期位相を基準とした客観世界である。

5.2 movePlayer() は動的 topoState を持つ主観空間を歩く

一方、プレイヤーの移動は現在の topoState を引数に渡す：

const result = topology.resolveStep(
  currentNode.x, currentNode.y, dir,
  playerState.topoState,  // 動的な位相状態
  graph.baseMap.width, graph.baseMap.height
);
playerState.topoState = result.topoState;  // 更新

プレイヤーは、同じ nodeId の世界を土台にしつつ、現在の rot によって接続の読み方が変わる主観位相付き空間を歩いている。

5.3 buildLocalChart() は主観的被覆空間の断面を描く

局所チャートは BFS で生成されるが、各ノードは nodeId・localX・localY・topoState の4つ組として展開される。同じ nodeId であっても、別の topoState・別の局所座標で再登場しうる。

この意味で局所チャートは、nodeId の単なる近傍図ではなく、主観位相付き被覆空間の一断面に近い。前稿§3の離散ファイバーバンドルにおける「局所自明化」の具体的実装である。

6. NPCはプレイヤーと同じ空間を歩いていない

6.1 NPCは静的 worldGraph を使って移動する

犬の追跡 updateDog() や猫の徘徊 updateCat() は、いずれも静的な game.worldGraph 上で距離計算・経路探索を行う：

// 犬の追跡
const playerDist = findDistance(
  game.worldGraph, dog.nodeId,
  game.playerState.nodeId, new Set(), 3
);
const nextNodeId = findNextStepToward(
  game.worldGraph, dog.nodeId, targetNodeId, blocked
);

// 猫の徘徊
const neighbors = getValidNeighborNodeIds(game.worldGraph, cat.nodeId)
  .filter(id => id !== game.playerState.nodeId);

ここで重要なのは、NPCが使うのは静的に構築された worldGraph であり、プレイヤーの現在 rot を共有していないことである。

6.2 二つの空間のずれ

プレイヤーは動的位相空間を歩き、犬や猫は静的世界骨格で行動する——同じ座標にいても、同じ空間にいるとは限らない。

7. 空間の三重分裂：移動・接触・描画

ここが本稿の核心である。通常のゲームでは、「隣接している」という概念は暗黙に複数の意味を共有している。

7.1 三つの隣接グラフ

グラフ
意味
実装上の根拠
移動グラフ $G_{\text{move}}$
1歩で到達できる
resolveStep() + 動的 topoState
接触グラフ $G_{\text{touch}}$
当たり判定が成立する
nodeId 一致判定
描画グラフ $G_{\text{draw}}$

普通の空間ではこの3つはほぼ一致する。しかし rot が入ると、それぞれが別のグラフに従ってしまう。

7.2 接触判定は nodeId ベースである

Topologue の接触判定は、移動先の nodeId が NPC の nodeId と一致するかで決まる：

// プレイヤーが犬に触れる
if (game.actors.dog && nextNodeId === game.actors.dog.nodeId) {
  petActor(game, "dog");
}

// 犬がプレイヤーに触れる
if (nextNodeId === game.playerState.nodeId) {
  actorTouchesPlayer(game, "dog");
}

一方、描画はプレイヤーの topoState 付き局所チャートを通して行われる。よって同じ nodeId に犬がいても：

  • 接触空間では「いる」 → 判定成立
  • 描画空間では「見えない」 → 局所チャートに現れない

ということが起こりうる。

8. 非対称隣接空間：一方通行の隣接

8.1 一方通行隣接の発生

rot をプレイヤーだけが持ち、NPCが共有しないとき、以下の事態が生じる：

  • 犬から見ると（worldGraph 上で）プレイヤーは隣接ノードにいる
  • プレイヤーから見ると（動的 topoState に基づく接続で）犬はまったく別の方向にいる

通常の隣接は対称関係 $A \sim B \Rightarrow B \sim A$ である。しかしここでは：

$$ A \to B \quad \text{かつ} \quad B \not\to A $$

が起こる。隣接関係が無向グラフではなく有向グラフになっている。

8.2 見えないが触れられる存在

この構造の結果として：

  • 犬は worldGraph 上でプレイヤーに接近・到達できる
  • プレイヤーの描画空間（局所チャート）には犬の姿が現れない
  • しかし nodeId 一致による接触判定は成立する

「見えないが触れられる敵」が、実装上の自然な帰結として発生する。

8.3 蜃気楼としての自己像

局所チャートは同じ nodeId が異なる topoState・異なる局所座標で再出現しうる構造を持つ。そのため、位相状態の違いが描画層へ漏れると：

  • 遠方に自分の別像が見える
  • 自分が別の存在に接触されているような像が現れる
  • 実際にはそこに自分はいない

これは「バグった残像」というより、異なる位相層をかすかに覗き見ている現象——被覆空間の別のシートに射影された自分の「リフト」を観測していると理解できる。

9. 非可換性と離散曲率

9.1 交換子としての離散曲率

東端通過作用 $g_E$ と北端通過作用 $g_N$ が可換でないとき：

$$ g_E \circ g_N \ne g_N \circ g_E $$

矩形ループに沿う交換子

$$ [g_E, g_N] = g_E \circ g_N \circ g_E^{-1} \circ g_N^{-1} $$

が恒等でなければ、そこに離散的な曲率が存在する。前稿§7で予想した離散ガウス・ボネの文脈でいえば、この交換子は「面の局所ホロノミー」に他ならない。

9.2 主体ごとの経路差が一方通行隣接を生む

非可換な接続のもとでは、経路が結果を決める。プレイヤーと犬が異なる経路でたどったとき、たとえ同じ座標に到達しても内部状態が一致しない：

  • プレイヤー系：犬は位相層の彼方にいる → 不可視
  • 犬の系：プレイヤーは接触可能位置にいる → 接近する

この食い違いは「ランダムなバグ」ではなく、非可換ホロノミーが主体ごとの状態差として現れたものである。

犬には追われているのに犬は見えない。自分の蜃気楼だけが別の層に見える。——この怪異は、大変まっとうな数学的帰結である。

10. セーブ/ロードによる位相差

補足として、セーブ/ロードの実装上の問題にも触れておく。

地形・座標は保存されるが、ラップ規則や位相モードだけが保存漏れするケースがある。すると、ロード後に：

  • 見た目は同じ
  • 現在地も同じ
  • しかし世界の貼り合わせだけが変わっている

「宇宙の微妙なズレ」が発生する。これは前稿の接続（ファイバーバンドルの接続）のみが失われ、底空間とファイバーの直積に退化したことに相当する。

11. まとめ

11.1 本稿で明らかにしたこと

  1. rot はホロノミー変数である。見た目の回転ではなく、境界通過履歴の蓄積として機能する。
  2. 真世界と主観空間は分離している。buildWorldGraph()（客観）と buildLocalChart()（主観）が別の空間を構成する。
  3. NPCとプレイヤーは同一位相を共有していない。NPCは静的 worldGraph、プレイヤーは動的 topoState 付き空間を歩く。
  4. 移動・接触・描画の空間が分裂しうる。rot により三つの隣接グラフが一致しなくなる。
  5. 非対称隣接が自然に発生する。犬には追われているが犬は見えない——という怪異は、非可換ホロノミーの帰結である。

11.3 結語

ファミコンRPGの2Dマップは、表面的には素朴な格子である。しかしその内部では——端の貼り合わせ、状態付き遷移、主体依存の隣接、非可換な境界作用、離散曲率、異なる位相層の重なり——驚くほど豊かな構造が発生しうる。

古典的な2Dマップは単なる古い実装ではなく、離散的で、状態依存で、時に怪異を含んだ宇宙論の実験場である。

実装位相論とは、その継ぎ目を眺める学問である。
